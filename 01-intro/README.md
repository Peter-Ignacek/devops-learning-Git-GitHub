# Introduction – Version Control System (VCS)

## What is VCS?

A Version Control System (VCS) is a tool that helps track changes in files over time.

It allows you to:
- track history of changes
- revert to previous versions
- collaborate with other developers

---

## Types of VCS

### 1. Local VCS

Stores versions locally on your machine.

Example:
- RCS

Pros:
- simple
- no network needed

Cons:
- no collaboration
- risk of data loss

---

### 2. Centralized VCS (CVCS)

<img width="1346" height="661" alt="image" src="https://github.com/user-attachments/assets/20b0fe9a-03ca-43ff-abf9-f4747e1b02cc" />


Uses a central server where all versions are stored.

Examples:
- SVN
- CVS

Pros:
- easy collaboration
- single source of truth

Cons:
- server failure = problem
- requires network

---

### 3. Distributed VCS (DVCS)

<img width="1352" height="767" alt="image" src="https://github.com/user-attachments/assets/6872ff34-915e-4831-b186-c96322547983" />


Each user has a full copy of the repository.

Examples:
- Git (most popular)
- Mercurial

Pros:
- works offline
- fast
- no single point of failure

Cons:
- more complex at the beginning

---

## Why Git?

Git is a Distributed VCS.

Key advantages:
- fast and efficient
- supports branching
- widely used in DevOps and software development
